# Rolin Bio 网站部署指南（GitHub 版）

## 前置条件

- 已有 GitHub 账号
- 已有域名（用于绑定）
- 本地已安装 Git

---

## 方案一：GitHub Pages（最简单，免费）

### 第 1 步：在 GitHub 创建仓库

1. 登录 GitHub → 点右上角 `+` → **New repository**
2. 仓库名填：`rolinbio-website`（或任意名称）
3. 选 **Public**（GitHub Pages 免费版要求公开仓库）
4. **不要**勾选 "Add a README file"
5. 点 **Create repository**

### 第 2 步：推送代码到 GitHub

在本地 `rolinbio-website/` 目录下运行：

```bash
# 关联远程仓库（把 YOUR_USERNAME 换成你的 GitHub 用户名）
git remote add origin https://github.com/YOUR_USERNAME/rolinbio-website.git

# 推送
git push -u origin main
```

### 第 3 步：开启 GitHub Pages

1. 进入仓库页面 → **Settings** → 左侧栏 **Pages**
2. **Source** 选 **Deploy from a branch**
3. **Branch** 选 `main`，文件夹选 `/ (root)`
4. 点 **Save**
5. 等待 1-2 分钟，页面顶部会显示你的网站地址：
   `https://YOUR_USERNAME.github.io/rolinbio-website/`

### 第 4 步：绑定自定义域名

1. 在仓库根目录创建 `CNAME` 文件（内容只有一行你的域名，如 `www.rolinbio.com`）
2. 将 `CNAME` 提交推送到 GitHub
3. 去你的域名 DNS 管理后台，添加记录：

| 记录类型 | 主机记录 | 记录值 |
|---------|---------|--------|
| CNAME | www | YOUR_USERNAME.github.io |

4. GitHub 仓库 Settings → Pages → **Custom domain** → 输入 `www.rolinbio.com` → **Save**
5. 勾选 **Enforce HTTPS**（建议开启）

> ⚠️ GitHub Pages 不支持 Decap CMS 后台（需要 Netlify Identity 服务）。如需 CMS 功能，请用方案二。

---

## 方案二：GitHub + Netlify（推荐，功能完整）

### 第 1 步：推送代码到 GitHub

（同方案一的第 1、2 步）

### 第 2 步：连接 Netlify

1. 访问 https://app.netlify.com/ → 注册/登录（可直接用 GitHub 账号登录）
2. 点 **Add new site** → **Import an existing project**
3. 选择 GitHub → 授权并找到 `rolinbio-website` 仓库
4. 配置构建设置：
   - **Build command**：留空
   - **Publish directory**：`.`
5. 点 **Deploy** → 1 分钟内部署完成，Netlify 会给你一个 `xxx.netlify.app` 的临时地址

### 第 3 步：绑定自定义域名

1. Netlify Dashboard → 选你的站点 → **Domain settings** → **Add custom domain**
2. 输入你的域名（如 `rolinbio.com`）
3. Netlify 会给出 DNS 配置指引，去域名管理后台添加：

| 记录类型 | 主机记录 | 记录值 |
|---------|---------|--------|
| A | @ | 75.2.60.5 |
| CNAME | www | xxx.netlify.app |

4. DNS 生效后（通常几分钟到几小时），Netlify 自动启用 HTTPS

### 第 4 步：启用 CMS 后台（可选）

1. Netlify Dashboard → **Identity** → **Enable Identity**
2. Registration preferences → 选 **Invite only**
3. 点 **Invite users** → 输入你的邮箱 → 邮件中确认注册
4. **Identity** → **Services** → **Enable Git Gateway**
5. 访问 `https://你的域名/admin/` → 用邮箱登录 → 即可可视化编辑新闻

### 第 5 步：自动部署

之后每次 `git push` 到 GitHub `main` 分支，Netlify 自动触发部署，约 1 分钟更新上线。

## 日常更新流程

### 更新新闻内容

**方式 A（CMS 后台，推荐）：**
1. 访问 `https://你的域名/admin/`
2. 登录 → 在 News 集合中新增/编辑文章
3. 保存后自动提交到 GitHub 并触发部署

**方式 B（手动 Git）：**
1. 在 `content/news/` 下创建 `.md` 文件
2. `git add -A && git commit -m "Add news: xxx" && git push`
3. 部署平台自动更新

### 修改公司信息/页面内容

1. 编辑对应的 HTML 文件
2. `git add -A && git commit -m "Update: xxx" && git push`
3. 部署平台自动更新

## 方案对比

| 特性 | GitHub Pages | GitHub + Netlify |
|------|-------------|-----------------|
| 费用 | 免费 | 免费 |
| 自动部署 | 需手动 push | push 后自动 |
| CMS 后台 | ❌ | ✅ |
| 表单功能 | ❌ | ✅ |
| 自定义域名 | ✅ | ✅ |
| HTTPS | ✅ | ✅ |
| 部署速度 | 1-2 分钟 | 约 1 分钟 |

**建议：** 如果不需要 CMS 后台管理新闻，GitHub Pages 最省事。如果要可视化编辑新闻内容，用 Netlify。
